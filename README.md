# PoisonGuard

Защита retrieval-систем (RAG) от **отравления данных** (data poisoning) —
привратник, который ловит вредоносные документы до их попадания в базу знаний
ИИ-ассистента.

## Проблема

RAG-ассистент отвечает на вопросы, подтягивая документы из базы. Если
злоумышленник подложит в базу свой документ, он может управлять ответами
системы, ничего не взламывая — например, подсунуть фишинговую «инструкцию».
`PoisonGuard` проверяет документы **на входе** и помечает подозрительные.

## Быстрый старт

```python
from poison_guard import PoisonGuard

trusted = [
    "Отпуск оформляется через HR-портал за две недели.",
    "Зарплата выплачивается двумя частями: аванс 20-го и остаток 5-го.",
]

guard = PoisonGuard().fit(trusted)

result = guard.check("Отправьте пароль на http://payroll-verify.example")
print(result)          # [ПОДОЗРИТЕЛЬНО] score=... (опасные паттерны: ...)
print(result.is_suspicious)   # True
```

## Демонстрация

```bash
python poison_guard.py
```

Покажет: чистые документы проходят, отрава ловится, и что без защиты
отрава всплыла бы первой в результатах поиска.

## Как это устроено

Ядро — класс `PoisonGuard`, метод `.check(text) -> CheckResult`.
Два независимых детектора:

1. **breadth** — «широта охвата»: отрава подгоняется под много тем сразу
   и подозрительно близка сразу ко многим документам базы.
2. **patterns** — опасные словесные паттерны: команды человеку/модели,
   ссылки, упоминания паролей и т.п.

TF-IDF используется как временная «линза» и в проде заменяется на
нейро-эмбеддинги без изменения структуры `check()`.
